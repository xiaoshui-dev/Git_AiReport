# Git_AiReport 新增功能说明

> 本文档用于记录近期版本新增和优化的核心功能。

## 本次更新概览

本次更新重点围绕 **多语言体验**、**AI 生成速度**、**并发参数可配置** 和 **界面操作便捷性** 进行增强。

---

## 1. 多语言切换

### 功能说明

应用现在支持多国家语言切换，并会在首次启动时自动读取当前系统语言作为默认语言。

### 已支持语言

- 简体中文
- English
- 日本語
- 한국어

### 使用方式

- 在顶部生成工具栏中直接修改“语言”下拉框。
- 也可以在“设置”窗口中修改语言。
- 语言选择会保存在本地，下次打开应用会自动恢复。

---

## 2. 错误提醒跟随语言变化

### 功能说明

用户界面中的主要提醒、状态文案和错误信息现在会跟随当前语言切换。

### 已覆盖内容

- API Endpoint 未填写提示
- 模型名称未填写提示
- Git 仓库路径未选择提示
- Git 提交记录读取失败提示
- AI 生成失败提示
- API 连接测试失败提示
- 复制、导出、恢复工作区等状态提示

---

## 3. AI 生成语言跟随界面语言

### 功能说明

报告生成时，系统会把当前选择的语言写入 AI 提示词中，让 AI 尽量按当前语言输出内容。

### 影响范围

- 日报
- 周报
- 月报
- 项目总结
- 开发日志

---

## 4. 顶部工具栏新增快捷控件

### 新增控件

- **语言切换**：可以直接在生成工具栏中切换界面和生成语言。
- **并发数设置**：可以直接在生成工具栏中调整 AI 请求并发数。

### 优点

- 不需要打开设置窗口就能调整常用参数。
- 生成前可以快速切换语言或调整速度。
- 更适合频繁生成报告的使用场景。

---

## 5. AI 生成速度优化

### 优化内容

- 默认并发数从 **3** 提升到 **6**。
- 并发数上限从 **8** 提升到 **24**。
- 默认请求间隔从 **250ms** 调整为 **0ms**。
- 请求间隔改为“按 worker 错峰”逻辑，只在用户设置间隔时生效。
- 日报、周报、月报和开发日志生成都支持新的并发调度逻辑。

### 使用建议

- 如果模型服务稳定，可以适当提高并发数。
- 如果出现限流或请求失败，建议降低并发数，或增加请求间隔。
- 如果生成内容较长，可以适当降低并发，提升稳定性。

---

## 6. 设置窗口增强

### 新增或优化项

- 增加语言选择项。
- 并发数上限提升到 24。
- 增加并发参数提示，提醒高并发可能触发模型服务限流。
- 保留字数、风格、读者、关注点、额外提示词等原有配置。

## 7. 构建与打包说明

### 已验证

- `npm run typecheck` 已通过。
- `npm run build` 已通过。
- Windows 安装包已成功生成。

### 平台说明

- Windows 版本可在当前 Windows 环境直接构建。
- macOS 版本需要在 macOS 环境构建。
- Linux AppImage / snap / deb 建议在 Linux 或 WSL 环境构建，因为它们依赖 `chmod`、`mksquashfs`、`fpm` 等工具。
